# 주식 투자 에이전트 - 시스템 아키텍처

## 1. 개요

**목표:** 정보 수집 → 분석 → 전략 수립 → 자동 실행까지 전 과정을 자동화하는 AI 투자 에이전트

**범위:** 국내 주식(KRX) 우선, 이후 미국 주식(NYSE/NASDAQ) 확장  
**실행 환경:** 로컬 PC (Windows/macOS)

## 3. 컴포넌트 상세 설명

### 3.1 데이터 수집 (Data Collector)

| 수집 대상 | 데이터 종류 | 주기 |
|---------|-----------|------|
| 주가/거래량 | OHLCV, 시가총액, 외인/기관 동향 | 실시간/일별 |
| 재무제표 | 손익계산서, 대차대조표, 현금흐름표 | 분기/연간 |
| 공시 | 사업보고서, 주요 경영사항, 지분변동 | 실시간 |
| 뉴스 | 종목 관련 뉴스, 시황, 산업 동향 | 실시간 |
| 매크로 | 금리, 환율, 코스피/코스닥 지수 | 일별 |

### 3.2 데이터 저장 (Storage)

```
storage/
├── market_data/       # 주가 시계열 데이터 (SQLite or PostgreSQL)
├── fundamental/       # 재무제표 데이터
├── news/              # 뉴스 텍스트 + 임베딩
├── disclosures/       # DART 공시 원문
└── strategies/        # 생성된 전략 및 백테스트 결과
```

### 3.3 분석 엔진 (Analysis Agent)

분석은 큰 그림 → 작은 그림 순서로 4단계로 진행한다.
시장 국면이 나쁠 때는 차트 신호가 아무리 좋아도 가짜 신호일 수 있으므로,
**반드시 1단계 판단 결과가 이후 단계의 해석에 영향을 준다.**

```
[1단계] 시장 국면 파악
 - 입력: 지수(KOSPI/KOSDAQ), 매크로 지표(금리, 환율), 거래대금
 - 출력: 시장 상태 (상승장 / 하락장 / 횡보장)
 - 상승장이 아니면 이후 매수 시그널 강도를 자동으로 낮춤

[2단계] 섹터 흐름 파악
 - 입력: 섹터별 수익률, 외국인·기관 매매 동향
 - 출력: 주도 섹터, 소외 섹터 분류
 - 주도 섹터 종목만 다음 단계로 진행

[3단계] 종목 차트 분석
 - 입력: OHLCV, 기술적 지표(RSI/MACD/볼린저밴드), 차트 패턴
 - 출력: 차트 점수 (매수 적합도), 시그널 종류
 - 패턴 예: 골든크로스, 이중바닥, 저점 거래량 후 돌파 등

[4단계] 크로스체크 (재무 + 뉴스)
 - 입력: 재무제표(PER/PBR/ROE), 공시, 뉴스 감성
 - 출력: 종목 최종 점수, 매수/보류/제외 판정
 - 차트가 좋아도 재무 이상 징후나 부정 뉴스 있으면 제외
```

### 3.4 LLM 코어 (전략 수립 브레인)

- 4단계 분석 결과 전체를 종합해 투자 전략 제안
- 시장 국면에 따라 공격적/방어적 전략 자동 조정
  - 상승장 → 모멘텀 전략, 섹터 로테이션 전략
  - 횡보장 → 평균회귀 전략, 배당주 전략
  - 하락장 → 현금 비중 확대, 방어주 전략
- 전략의 근거와 리스크 요인 설명
- 종목 스크리닝 기준 제시

### 3.5 전략 에이전트 (Strategy Agent)

- LLM이 제안한 전략을 실행 가능한 규칙(Rule)으로 변환
- 매수/매도 조건 수식화
- 포지션 사이징 계산 (리스크 관리)
- 백테스터에 전략 전달 → 검증

### 3.6 백테스트 엔진 (Backtester)

- 과거 데이터 기반 전략 시뮬레이션
- 성과 지표 계산: 수익률, 샤프비율, 최대낙폭(MDD), 승률
- 전략 파라미터 최적화 (그리드 서치)
- 결과를 LLM에 피드백 → 전략 개선 루프

### 3.7 실행 에이전트 (Execution Agent)

- 검증된 전략의 시그널 감지
- 증권사 API를 통한 실제 주문 실행
- 주문 유형: 시장가/지정가 선택
- 체결 확인 및 포지션 추적
- 손절/익절 조건 모니터링

### 3.8 알림 시스템 (Notifier)

- 텔레그램 봇: 매수/매도 시그널, 체결 알림
- 일간 리포트: 포트폴리오 현황, 손익, 전략 평가
- 이상 감지 알림: 큰 손실 발생 시 즉시 알림

## 5. 에이전트 간 통신

```python
# 컴포넌트 간 데이터 교환 기본 구조 (예시)

# 분석 결과 객체
AnalysisResult = {
    "timestamp": "2024-01-15T09:00:00",
    "market_phase": "uptrend",       # 시장 국면
    "sector_rotation": ["반도체", "2차전지"],  # 주목 섹터
    "risk_level": "medium",          # 리스크 레벨
    "indicators": {...},             # 기술적 지표 요약
    "fundamental_signals": {...}     # 재무 신호
}

# 전략 객체
Strategy = {
    "id": "strategy_001",
    "name": "모멘텀 + 재무건전성",
    "type": "momentum",
    "buy_conditions": [...],         # 매수 조건 리스트
    "sell_conditions": [...],        # 매도 조건 리스트
    "position_size": 0.1,           # 종목당 비중 (10%)
    "max_positions": 5,             # 최대 보유 종목 수
    "stop_loss": -0.05,             # 손절 기준 (-5%)
    "take_profit": 0.15,            # 익절 기준 (+15%)
    "backtest_result": {...}         # 백테스트 결과
}
```

## 6. 시스템 실행 모드

| 모드 | 설명 | 사용 시점 |
|-----|------|---------|
| `research` | 데이터 수집 + 분석만, 매매 없음 | 개발 초기, 전략 탐색 |
| `backtest` | 과거 데이터로 전략 시뮬레이션 | 전략 검증 |
| `paper` | 가상 머니로 실전 시뮬레이션 | 실전 전 최종 검증 |
| `live` | 실제 자금으로 자동매매 | 검증 완료 후 |

**반드시 research → backtest → paper → live 순서로 진행**
