# 주식 투자 에이전트 - 프로젝트 정의서

## 1. 한 줄 정의

> **"투자 경험 없이도 데이터 기반으로 국내 주식에 투자할 수 있게 해주는 AI 에이전트"**

---

## 2. 문제 정의 (Why)

### 개인 투자자가 겪는 현실적인 어려움

**정보 비대칭 문제**
- 전문 기관투자자는 수십 명의 애널리스트가 매일 수백 개 종목을 분석
- 개인 투자자 혼자 모든 종목의 뉴스, 공시, 재무제표를 따라가기 불가능
- 좋은 투자 기회를 뒤늦게 발견하거나 아예 놓침

**감정적 판단의 문제**
- 주가가 오르면 욕심에 추격 매수, 내리면 공포에 손절
- 충분한 분석 없이 뉴스 헤드라인이나 커뮤니티 글에 영향받아 매매
- 손실 회피 편향으로 손절을 미루다 손실 확대

**전략 부재 문제**
- 명확한 매수/매도 기준 없이 '느낌'으로 투자
- 어떤 상황에서 어떤 전략이 유효한지 체계적으로 모름
- 한 번 실패하면 전략을 바꾸지만 새 전략도 근거 없음

### 이 프로젝트가 해결하려는 것

| 문제 | 에이전트의 해결책 |
|-----|---------------|
| 정보 비대칭 | 수백 종목 데이터를 자동 수집·분석, 중요한 것만 요약 |
| 감정적 판단 | 사전에 정의된 규칙(전략)대로만 매매, 감정 개입 차단 |
| 전략 부재 | 시장 상황에 맞는 전략을 AI가 제안하고 백테스트로 검증 |

---

## 3. 목표 (What)

### 최종 목표 (End Goal)
투자 판단의 전 과정을 자동화한 에이전트:
1. **정보 수집**: 주가, 재무제표, 공시, 뉴스를 자동으로 수집
2. **분석**: 수집된 데이터를 해석해 시장 상황과 종목 상태 파악
3. **전략 수립**: 현재 시장에 맞는 투자 전략을 AI가 제안하고 백테스트 검증
4. **실행**: 검증된 전략으로 자동 매수/매도

### 단계별 목표

| 단계 | 목표 | 의미 |
|-----|------|------|
| 1단계 | 데이터 자동 수집 | "내가 직접 찾아보지 않아도 된다" |
| 2단계 | AI 시장 분석 리포트 | "오늘 시장이 어떤지, 뭘 봐야 하는지 알 수 있다" |
| 3단계 | 전략 제안 + 검증 | "이 전략이 과거에 통했는지 확인하고 투자한다" |
| 4단계 | 자동 실행 | "설정해두면 알아서 사고 판다" |

## 4. 성공 기준 (Success Metrics)

### 정량적 기준

| 지표 | 기준 | 측정 방법 |
|-----|------|---------|
| 수익률 | KOSPI 수익률 이상 (벤치마크 초과) | 연간 포트폴리오 수익률 비교 |
| 최대낙폭(MDD) | -15% 이내 | 운용 기간 최대 손실 |
| 에이전트 가동률 | 장 운영일 99% 이상 정상 동작 | 오류 없이 데이터 수집·분석 |
| 전략 승률 | 50% 이상 | 매매 건 중 수익 비율 |

### 정성적 기준
- 매일 아침 시장 분석 리포트를 받아 "오늘 뭘 봐야 할지" 알 수 있는 상태
- 어떤 이유로 매수/매도했는지 근거가 항상 남아있는 상태
- 감정적으로 충동 매매를 하고 싶은 순간에 에이전트의 기준을 믿고 기다릴 수 있는 상태

---

## 5. 범위 (Scope)

### In Scope (이 프로젝트에서 할 것)
- 국내 주식(KOSPI, KOSDAQ) 대상
- 중장기 투자 (보유 기간 수 일 ~ 수 주 기준)
- KOSPI 200 주요 종목 대상 (상위 200개 종목)
- 로컬 PC에서 실행되는 에이전트

### Out of Scope (이 프로젝트에서 안 할 것)
- 선물, 옵션, ETF, 채권 등 파생상품
- 초단타(HFT) 또는 당일 매매(데이트레이딩)
- 미국 주식 (Phase 1 완료 후 별도 확장)
- 타인에게 투자 조언을 제공하는 서비스화

## 6. 제약 조건 (Constraints)

| 제약 | 내용 |
|-----|------|
| **예산** | API 비용 월 5만원 이내 유지 목표 |
| **실행 환경** | 개인 PC (24시간 상시 가동 아님, 장 시간대만 실행) |
| **법적 제약** | 금융투자업 미등록 → 타인 자금 운용 불가, 본인 자금만 |
| **초기 투자금** | 소액으로 시작 (실전 전환 시 50~100만원 테스트) |
| **데이터 제약** | 무료 API 위주 사용 (실시간 호가 데이터는 유료) |

## 7. 리스크 (Risks)

| 리스크 | 가능성 | 영향 | 대응 방안 |
|-------|-------|------|---------|
| 에이전트 오류로 의도치 않은 대량 주문 | 중 | 높음 | 1회 주문 한도 및 일일 손실 한도 하드코딩 |
| 과최적화 (백테스트에서만 좋은 전략) | 높음 | 중간 | 워크포워드 테스트, 모의투자 필수 선행 |
| API 서비스 중단 (증권사, DART 등) | 낮음 | 중간 | 에러 시 알림 + 자동 거래 중단 |
| 블랙스완 이벤트 (시장 급락) | 낮음 | 높음 | 최대낙폭 기준 초과 시 전량 매도 |
| LLM 환각(Hallucination)으로 잘못된 전략 | 중 | 중간 | 전략은 반드시 백테스트 통과 후 실행 |

## 9. 프로젝트 철학

> **"AI가 분석하고, 규칙이 실행하고, 사람은 감독한다"**

- AI는 전략을 제안하지만, 사람이 최종 활성화 여부를 결정
- 한 번 활성화된 전략은 감정 개입 없이 규칙대로만 실행
- 모든 판단의 근거는 기록되고, 언제든 검토 가능해야 함
- 수익보다 손실 방어를 우선: 잃지 않으면 기회는 다시 온다
