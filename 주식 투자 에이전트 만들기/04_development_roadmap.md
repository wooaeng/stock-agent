# 주식 투자 에이전트 - 단계별 개발 로드맵

## 개발 원칙

1. **단계별 검증**: 각 단계 완료 후 충분히 검증 후 다음 단계 진행
2. **실제 돈은 마지막**: 4단계(자동매매) 전까지 절대 실제 자금 투입 안 함
3. **리스크 우선**: 모든 기능에 안전장치 먼저 구현
4. **데이터 중심**: 좋은 데이터 없이 좋은 전략 없음

---

## 전체 일정 개요

```
Phase 1: 데이터 파이프라인    [예상: 3~4주]
Phase 2: 분석 + LLM 전략 수립  [예상: 4~6주]
Phase 3: 백테스트 + 검증       [예상: 3~4주]
Phase 4: 자동매매 실행         [예상: 4~6주]
                               ─────────────
                               총 14~20주 (약 3~5개월)
```

> 개발 경험 및 투입 시간에 따라 크게 달라질 수 있음.  
> 각 단계를 서두르지 말고 충분히 완성 후 다음으로 넘어갈 것.

---

## Phase 1: 데이터 파이프라인 구축

**목표:** 신뢰할 수 있는 데이터를 자동으로 수집하고 저장하는 시스템

**기간:** 3~4주

### 1.1 환경 세팅 (1~2일)
- [ ] Python 가상환경 구성
- [ ] 프로젝트 디렉토리 구조 생성
- [ ] API 키 발급 및 환경변수 설정
  - 한국투자증권 Open API (모의투자 계좌 개설 포함)
  - DART Open API
  - 텔레그램 봇 토큰

### 1.2 시장 데이터 수집기 (3~5일)
- [ ] `pykrx` 연동: 일별 OHLCV 수집
- [ ] KOSPI 200 종목 리스트 관리
- [ ] SQLite DB 스키마 생성 및 초기 데이터 적재 (5년치)
- [ ] 투자자별 매매 동향 수집

### 1.3 재무/공시 데이터 수집기 (4~5일)
- [ ] DART API 연동: 재무제표 수집
- [ ] 공시 목록 수집 및 저장 (30분 간격 폴링)
- [ ] 공시 원문 텍스트 추출

### 1.4 뉴스 수집기 (2~3일)
- [ ] 네이버 금융 뉴스 크롤러 구현
- [ ] 종목별 뉴스 매핑
- [ ] 중복 제거 로직

### 1.5 매크로 데이터 수집기 (1~2일)
- [ ] 한국은행 ECOS API 연동
- [ ] 환율, 금리, 지수 데이터 수집

### 1.6 스케줄러 구성 (1~2일)
- [ ] APScheduler로 수집 자동화
- [ ] 장 운영일/휴장일 처리
- [ ] 수집 실패 시 재시도 로직

### 1.7 검증 (2~3일)
- [ ] 데이터 무결성 검사 (누락, 이상값 탐지)
- [ ] 수집 로그 확인
- [ ] 1주일 자동 수집 정상 동작 확인

**Phase 1 완료 기준:**
- KOSPI 200 종목의 5년치 데이터가 DB에 적재됨
- 매일 자동으로 새 데이터가 수집됨
- 수집 실패 시 텔레그램 알림 발송됨

## Phase 2: 분석 엔진 + LLM 전략 수립

**목표:** 수집된 데이터를 분석하고 Claude API로 투자 전략을 생성하는 시스템

**기간:** 4~6주

### 2.1 기술적 분석 모듈 (3~4일)
- [ ] `pandas-ta`로 지표 계산 자동화
  - RSI, MACD, 볼린저밴드, 이동평균, 거래량 비율
- [ ] 계산된 지표 DB 저장
- [ ] 종목 스크리닝 함수 (예: RSI < 30 이면서 거래량 증가)

### 2.2 재무 분석 모듈 (3~4일)
- [ ] 주요 밸류에이션 지표 계산 (PER, PBR, ROE 등)
- [ ] 재무 건전성 점수 계산 로직
- [ ] 섹터별 평균 대비 비교 기능

### 2.3 뉴스 감성 분석 (2~3일)
- [ ] Claude API로 뉴스 감성 분류 (긍정/중립/부정)
- [ ] 종목별 감성 점수 집계
- [ ] 이상 감성 감지 (급격한 부정 뉴스 등)

### 2.4 시장 국면 판단 모듈 (2~3일)
- [ ] 추세 판단 로직 (상승장/하락장/횡보장)
- [ ] 섹터 로테이션 감지
- [ ] VIX 대용 지표 계산 (변동성 지수)

### 2.5 LLM 입력 포맷터 (3~4일)
- [ ] 분석 결과를 LLM 프롬프트로 변환하는 템플릿 설계
- [ ] 토큰 수 최적화 (핵심 정보만 압축)
- [ ] 구조화된 JSON 출력 스키마 정의

### 2.6 전략 수립 에이전트 (5~7일)
- [ ] Claude API 연동
- [ ] 전략 생성 프롬프트 작성 및 튜닝
- [ ] LLM 출력을 구조화된 전략 객체로 파싱
- [ ] 전략 저장 및 이력 관리
- [ ] 전략 설명 리포트 자동 생성

### 2.7 텔레그램 리포트 (2~3일)
- [ ] 일간 시장 분석 리포트 발송
- [ ] 새 전략 생성 시 요약 알림
- [ ] 주목 종목 리스트 발송

**Phase 2 완료 기준:**
- 매일 오전 시장 분석 + 투자 전략이 자동 생성됨
- 생성된 전략이 텔레그램으로 발송됨
- 전략이 DB에 저장되고 이력 조회 가능

## Phase 3: 백테스트 및 전략 검증

**목표:** 생성된 전략의 과거 성과를 검증하고 전략을 개선하는 시스템

**기간:** 3~4주

### 3.1 백테스트 엔진 구축 (5~7일)
- [ ] `vectorbt` 기반 백테스트 프레임워크 구현
- [ ] 전략 객체 → 백테스트 변환 어댑터
- [ ] 거래 비용 반영 (수수료 0.015%, 세금 0.2%)
- [ ] 슬리피지 모델 적용

### 3.2 성과 지표 계산 (2~3일)
- [ ] 누적 수익률, 연환산 수익률
- [ ] 최대낙폭 (MDD)
- [ ] 샤프비율, 소르티노비율
- [ ] 승률, 평균 손익비
- [ ] 벤치마크(KOSPI) 대비 초과수익

### 3.3 전략 개선 루프 (5~7일)
- [ ] 백테스트 결과를 LLM에 피드백하는 프롬프트 설계
- [ ] 파라미터 최적화 (그리드 서치)
- [ ] 전략 버전 관리 (v1 → v2 → v3)
- [ ] 워크포워드 분석 (과최적화 방지)

### 3.4 종목 스크리닝 시스템 (3~4일)
- [ ] 전략 조건에 맞는 종목 자동 필터링
- [ ] 매수 우선순위 산정 (스코어링)
- [ ] 포트폴리오 구성 시뮬레이션

### 3.5 모의투자 연동 (3~4일)
- [ ] 한국투자증권 모의투자 계좌 연동
- [ ] 실시간 시그널 → 모의 주문 실행
- [ ] 모의 포트폴리오 성과 추적 (최소 2주 운용)

### 3.6 대시보드 구현 (3~4일)
- [ ] Streamlit 기반 모니터링 화면
- [ ] 전략별 성과 비교 차트
- [ ] 실시간 포트폴리오 현황

**Phase 3 완료 기준:**
- 생성된 전략의 백테스트 결과가 자동으로 계산됨
- 성과가 기준 이하인 전략은 LLM이 자동 개선 제안
- 모의투자 2주 이상 정상 동작 확인
- MDD < 15%, 샤프비율 > 0.5인 전략 최소 1개 확보

## Phase 4: 실제 자동매매 실행

**목표:** 검증된 전략으로 실제 자금 자동 투자 (소액으로 시작)

**기간:** 4~6주

> **⚠️ 경고:** 이 단계부터 실제 금전 손실 가능. Phase 3 완료 및 충분한 검증 후 진행.

### 4.1 실전 계좌 연동 (3~4일)
- [ ] 한국투자증권 실전 API 전환
- [ ] 실전/모의 모드 스위치 구현
- [ ] API 오류 처리 및 재연결 로직

### 4.2 리스크 관리 모듈 (5~7일)
- [ ] 일일 최대 손실 한도 (예: -3% 도달 시 자동 거래 중단)
- [ ] 종목당 최대 비중 제한 (예: 전체 자금의 20% 이하)
- [ ] 최대 보유 종목 수 제한
- [ ] 급등락 종목 자동 제외
- [ ] 비상 전체 매도 기능

### 4.3 주문 실행 모듈 (3~4일)
- [ ] 지정가/시장가 주문 로직
- [ ] 분할 매수/매도 (한 번에 전량 매매 방지)
- [ ] 체결 확인 및 미체결 처리
- [ ] 주문 이력 DB 저장

### 4.4 포지션 관리 (3~4일)
- [ ] 실시간 포지션 추적
- [ ] 손절/익절 자동 실행
- [ ] 트레일링 스탑 (수익 보호)

### 4.5 운영 안정화 (7~14일)
- [ ] 소액(50~100만원)으로 1~2주 실전 운영
- [ ] 오류 모니터링 및 수정
- [ ] 점진적 투자금 확대

**Phase 4 완료 기준:**
- 실전 2주 이상 무장애 자동 운행
- 모든 거래가 DB에 기록됨
- 리스크 한도 초과 시 자동 중단 동작 확인
- 텔레그램 알림 모두 정상 작동
